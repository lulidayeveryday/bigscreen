数据资产
Data assets

资产信息采集
数据资产将与元数据、数据质量、ETL等模块通过接口互通，根据需求获取数据资产实体信息和数据资产监控指标，实现资产跨平台串联，增强资产在监控运维和价值评估中的数据协同
资产注册
基于元数据采集的对象，进行资产的自动标识以及手动补全，完善资产的业务和管理信息，通过数据资产注册赋予资产唯一性和权威性，以使资产能够在企业范围内或更大范围内得到认知
资产生命周期管理
对数据资产建立从注册到下线的全过程生命周期管理，监控数据资产的变更和使用情况，对数据资产何时需要重点治理、何时需要版本变更、何时需要下线释放资源做出合理的预警提示

资产变更管理
构建自动化的资产采集、识别和分析流程，发现资产变更，并将变更产生的影响提前通知到资产使用者
资产分析评估
针对企业全域资产进行多维度多视角评估，直观展示资产现状。统计数据评估结果，进行趋势分析，根据资产的评估走势给出正向的指引或对风险的预警。通过权重计算资产六维评估价值分数，进行纵横比对和总体分布分析
资产安全管理
数据资产分层分级管控、敏感资产自动识别、资产发布审核、资产查阅申请审核

统一监控
Operation Monitoring

采集监控
以数据源为入口，提供对文件采集入库的监控，包括入库情况、开始时间、结束时间等

加工监控
以数据目标对象为入口，提供加工明细监控，包括加工流程、流程执行情况、起止时间等。

稽核监控
以实体对象为入口，提供稽核明细监控，包括稽核结果、稽核规则、执行状态、错误级别等。

血缘监控
以数据地图为入口，提供全局监控，监控每个对象的数据流转、稽核情况。
